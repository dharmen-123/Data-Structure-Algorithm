
    // // ****** Searching and Searching Algorithms ****** // //

A searching algorithm is a method used to find the location of a target element in a data structure (like an array, list, or database).
It answers the question: “Does the element exist, and if yes, where?”

***  Importance of Searching ***

Data retrieval: Quickly find records in databases.
Optimization: Used in pathfinding, AI, and problem-solving.
Foundation: Many algorithms (like sorting, graph traversal) rely on searching.

***  Real-Life Examples: ***

Searching a contact in your phone.
Looking up a word in a dictionary.
Finding a product in an online store.

    **** Types of Searching Algorithms ****

1. Linear Search
    Idea: Check each element one by one until the target is found or list ends.

Example:  
    Array = {10, 20, 30, 40}, Target = 30 → Found at index 2.

Complexity:
    Best Case: O(1) (first element)
    Worst Case: O(n) (last element or not found)
    Average Case: O(n)
Use: Works on unsorted data, but inefficient for large datasets.

2. Binary Search

    Idea: Works on sorted arrays. Repeatedly divide the search interval in half.
Steps:
    Compare target with middle element.
    If equal → found.
    If smaller → search left half.
    If larger → search right half.

Example:  
    Array = {10, 20, 30, 40, 50}, Target = 30 → Found at index 2.

Complexity:
    Best Case: O(1) (middle element)
    Worst Case: O(log n)
    Average Case: O(log n)
    Use: Very efficient for large sorted datasets.

3. Jump Search

Idea: Works on sorted arrays. Jump ahead by fixed steps (√n), then do linear search in that block.
    Complexity: O(√n).
    Use: Faster than linear search, but slower than binary search.

4. Interpolation Search

Idea: Improvement over binary search for uniformly distributed data.
Uses formula to estimate position of target.
Complexity:
    Best Case: O(log log n)
    Worst Case: O(n)
    Use: Searching in databases with uniform distribution (like student roll numbers).

5. Exponential Search

Idea: Useful for unbounded or infinite lists.
Steps: Find range by repeated doubling, then apply binary search.
    Complexity: O(log n).
    Use: Searching in infinite or very large datasets.
